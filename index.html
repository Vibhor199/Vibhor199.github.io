Onboarding API

  • After some analysis of user data, i found that ML can be used to help people fill their details on our marriage portal. I was given the task of fully developing this feature. To help reduce onboarding time which can be huge for marriage portals.
  • Completed the development of flask based apis using best practices like PEP8, pylint in Pycharm.
  • Used ORM sqlalchemy and Marshmallow as serializer.
  • These API's are to run ML Models on small amount of user data like age and name and use it to get religion, mother_tongue and most likely partner preferences of the user.
  • Learned about scalable deployment architecture and load distribution among processing nodes. Learned Redis for key value in-memory storage and Celery for job scheduling.
  • Used postman for saving and documenting the API and learned PyTest to write tests for them. Also learned how these thing can prevent issues in long run.
  • Models developed were in Tensorflow and SciKitLearn for the above APIs. LSTM based RNN, Random Forest, K Nearest Neighbors were some of the models used.
  • Result : Reduced onboarding to few clicks. User retention was increased.
